Finding

P2 — “Voice settings” appears to do nothing from Reader Settings

User impact: Tapping Voice settings in the reader’s settings panel leaves the panel on screen. The Voice & Speech screen has rendered behind it, but is invisible until the user manually presses Cancel. This looks like a failed tap and makes the destination difficult to discover.

Reproduction: Open a downloaded chapter → overflow → Reader settings → Voice settings. The modal remains. Press Cancel and the Voice & Speech page is then visible.

Evidence: The panel row calls showTtsSettings() but does not dismiss its AlertDialog. The dialog is created and only the Cancel button dismisses it in android/app/src/main/java/com/vinicius741/webnovelarchiver/ui/ReaderPanels.kt.

Suggested fix: Give the panel-row callback a way to dismiss the dialog before changing screens, or make “Voice settings” close the panel explicitly and then invoke showTtsSettings(). Add a UI-level regression test or manual QA case that asserts the destination is immediately visible after the tap.

Usability observations

  • The library, details, and reader surfaces were visually coherent at the Pixel 8 portrait size: no clipped app-bar content, broken cover rendering, or contrast problem was observed.
  • The reader’s fixed chapter-navigation bar can temporarily cover the lowest visible line of the WebView while reading. The text can be scrolled clear, so this is recorded as a minor polish opportunity rather than a confirmed loss-of-content bug. Consider adding enough bottom document padding to keep the last visible line above the bar.
  • The compact empty states for the download queue and updates page were clear and gave an actionable next step.

Not exercised in this session

  • Live Royal Road/Scribble Hub import and sync, including Cloudflare handling: these depend on current third-party behavior and can add or modify stories.
  • Downloading new chapters from a remote source and queue pause/resume/error recovery: the existing queue was empty, and starting downloads would modify the emulator’s library.
  • Opening the generated EPUB in an external reader and backup/restore/file-picker workflows.
  • Landscape, folded/unfolded, accessibility screen-reader, and network-loss scenarios.

Recommended next steps

  1. Fix the reader-settings modal dismissal before navigation (P2).
  2. Run a separate network-enabled regression pass for source import, sync, downloads, and Cloudflare challenge recovery using disposable test stories.
  3. Run posture and accessibility checks after the navigation fix, especially for reader controls and long details pages.
